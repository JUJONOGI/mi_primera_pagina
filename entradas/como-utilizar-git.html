Introducción

Git es una herramienta esencial para el control de versiones en el desarrollo de software. En esta guía, aprenderás los conceptos básicos para empezar a utilizar Git de manera efectiva.

1. Inicialización del Repositorio Local

Para comenzar a usar Git en tu proyecto, primero debes inicializar un repositorio local. Esto se hace utilizando el comando git init.

git init

Este comando crea un subdirectorio oculto llamado .git que contiene todos los archivos necesarios para el control de versiones.

Ejemplo Práctico:

cd mi-proyecto
git init

2. Creación de Branches

Las ramas (branches) en Git permiten trabajar en diferentes funcionalidades o correcciones sin afectar la rama principal (main). Esto facilita el desarrollo colaborativo y el manejo de versiones.

Crear una Nueva Rama:

git branch nombre-de-la-rama

Cambiar de Rama:

git checkout nombre-de-la-rama
# o usando git switch
git switch nombre-de-la-rama

Unir Ramas con Merge:

git checkout main
git merge nombre-de-la-rama

Ejemplo Práctico:

git branch feature/nueva-funcionalidad
git checkout feature/nueva-funcionalidad

# Realizar cambios en el código

git add .
git commit -m "Agregar nueva funcionalidad"
git checkout main
git merge feature/nueva-funcionalidad

3. Agregar un Repositorio Remoto

Conectar tu repositorio local a un repositorio remoto en GitHub te permite respaldar tu código y colaborar con otros desarrolladores.

Agregar el Repositorio Remoto:

git remote add origin https://github.com/JUJONOGI/mi_primera_pagina.git

Enviar Cambios al Repositorio Remoto:

git push -u origin main

Ejemplo Práctico:

git remote add origin https://github.com/JUJONOGI/mi_primera_pagina.git
git push -u origin main

Conclusión

En esta guía, has aprendido los fundamentos básicos para utilizar Git en tus proyectos. Desde la inicialización de un repositorio local hasta la creación de ramas y la conexión con un repositorio remoto, estos conceptos te ayudarán a gestionar de manera efectiva el control de versiones en tus desarrollos.
